# Hieco Mirror CLI

Use this skill when the task involves the `hieco` CLI or `@hieco/mirror-cli`.

## Workflow

1. Start with [references/command-selection.md](references/command-selection.md).
2. Load [references/sources.md](references/sources.md) when you need GitHub links, npm links, or installed `node_modules` lookup paths.
3. Load [references/installation.md](references/installation.md) for install and invocation patterns.
4. Load [references/quickstart.md](references/quickstart.md) for copy-paste examples.
5. Load [references/best-practices.md](references/best-practices.md) for output, filtering, and scripting guidance.
6. Load [references/api-common.md](references/api-common.md) for global flags, output behavior, ID formats, and command conventions.
7. Load the domain API file that matches the task:
   - accounts and balances: [references/api-accounts.md](references/api-accounts.md)
   - tokens and NFTs: [references/api-tokens.md](references/api-tokens.md)
   - transactions: [references/api-transactions.md](references/api-transactions.md)
   - blocks: [references/api-blocks.md](references/api-blocks.md)
   - contracts: [references/api-contracts.md](references/api-contracts.md)
   - schedules: [references/api-schedules.md](references/api-schedules.md)
   - topics: [references/api-topics.md](references/api-topics.md)
   - network: [references/api-network.md](references/api-network.md)
