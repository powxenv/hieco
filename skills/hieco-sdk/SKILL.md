# Hieco SDK

Use this skill when the task is about building with `@hieco/sdk` or `@hieco/react`.

Start here when the user wants Hedera reads, writes, signer-aware application logic, React hooks, or guidance on how the core Hieco app layer fits together.

## Start Here

1. Read [references/package-selection.md](references/package-selection.md) to choose the right package entry point.
2. Read [references/installation.md](references/installation.md) when the task needs setup or dependency guidance.
3. Read [references/quickstart.md](references/quickstart.md) for the shortest path to a working example.

## Pull In As Needed

Use these references only when the task calls for them:

- [references/frameworks.md](references/frameworks.md) for Next.js, SSR, hydration, or client-server boundary questions
- [references/best-practices.md](references/best-practices.md) for runtime, signer, and transaction guidance
- [references/api-sdk.md](references/api-sdk.md) for the core fluent client
- [references/api-react.md](references/api-react.md) for the React provider and hooks
- [references/api-appkit.md](references/api-appkit.md) for the legacy AppKit bridge
- [references/sources.md](references/sources.md) for repository, npm, and installed-package lookup paths

## Route To A Different Skill When Needed

- If the task is primarily about wallet connection, QR flows, extensions, or wallet UI, use `hieco-wallet`.
- If the task is primarily about read-only Mirror Node access, use `hieco-mirror`.
- If the task is primarily about live relay subscriptions, use `hieco-realtime`.
