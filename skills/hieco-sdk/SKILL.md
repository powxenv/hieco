# Hieco SDK

Use this skill when the task involves `@hieco/sdk` or `@hieco/react`.

## Workflow

1. Start with [references/package-selection.md](references/package-selection.md).
2. Load [references/sources.md](references/sources.md) when you need GitHub links, npm links, or installed `node_modules` lookup paths.
3. Load [references/installation.md](references/installation.md) for dependency setup.
4. Load [references/quickstart.md](references/quickstart.md) for copy-paste entry points.
5. Load [references/frameworks.md](references/frameworks.md) when the task mentions Next.js, TanStack Start, React Router framework mode, SSR, hydration, or client-server boundaries.
6. Load [references/best-practices.md](references/best-practices.md) for runtime, signer, and transaction guidance.
7. Load the API file that matches the package surface:
   - core fluent SDK: [references/api-sdk.md](references/api-sdk.md)
   - React provider and hooks: [references/api-react.md](references/api-react.md)
   - legacy AppKit bridge: [references/api-appkit.md](references/api-appkit.md)

If the task is primarily about wallet connection, wallet UI, QR, deeplinks, or `@hieco/wallet-react`, use the wallet skill instead.
